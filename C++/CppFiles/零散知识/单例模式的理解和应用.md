# 单例模式是什么

让某个类型在进程生命周期内**最多只存在一个实例**，并且提供**全局访问点**。典型用途：日志器、配置中心、线程池、度量上报器、全局缓存等。
# 为什么要这样实现（核心约束）

一个“可靠”的单例实现需要同时满足：

1. **唯一性**：外部不能复制/移动出第二个对象。
2. **线程安全**：第一次创建时多线程不抢跑；后续读取无锁或开销低。
3. **初始化顺序可控**：尽量避免“静态初始化顺序灾难”。
4. **生命周期可预期**：需要时能延迟创建；有时要避免在退出阶段被过早析构。
5. **易用与可测试**：最好能在测试里替换或重置实例。

下面按这些目标给出三种常用实现，并解释背后的原因。
## ① 推荐：Meyers Singleton（最简、最稳、线程安全）

**适用**：不需要自定义构造参数；退出时自动析构不敏感（或无所谓泄露）。
```C++
// C++11+
class Logger {
public:
    static Logger& instance() {
        // C++11 起，“函数内局部静态变量”的初始化是线程安全的
        static Logger inst;
        return inst;
    }

    // 禁止拷贝和移动，保证唯一实例
    Logger(const Logger&) = delete;
    Logger& operator=(const Logger&) = delete;
    Logger(Logger&&) = delete;
    Logger& operator=(Logger&&) = delete;

    void log(const std::string& msg) { /* ... */ }

private:
    Logger() { /* 打开文件/连接等 */ }
    ~Logger() = default; // 或者 ~Logger() = default;
};

```

**为什么这样写？**

- `static Logger inst;` 使用**“按需构造（construct-on-first-use）”**避免了全局/命名空间静态对象带来的**初始化顺序灾难**（不同翻译单元间谁先初始化不受控）。
- C++11 标准保证该局部静态的**第一次初始化是原子且只执行一次**，天然线程安全；后续获取是普通引用返回，零锁开销。
- `= delete` 禁止拷贝/移动，杜绝复制出第二份对象。

**析构问题**  
函数内静态对象会在程序结束时自动析构（顺序为**与其完成初始化的逆序**）。如果你担心退出阶段其它静态对象在析构时又访问到了这个单例，可考虑**不析构**（见下一个技巧）。
不析构的做法（“有意泄露”以换稳定）：

```C++
static Logger& instance() {
    static Logger* p = new Logger(); // 故意不 delete
    return *p;
}
```

在很多服务型进程里，OS 会回收进程内存，这样可彻底回避退出次序问题。
## ② 进阶：可传参/可重置/可替换（基于 `call_once` + `unique_ptr`）

**适用**：第一次创建需要外部参数（如配置路径）、需要手动销毁或在测试中替换实例。
```C++
#include <memory>
#include <mutex>
#include <string>

class Config {
public:
    // 第一次调用可传入构造参数，之后参数将被忽略
    static Config& instance(const std::string& path = "config.json") {
        std::call_once(init_flag_, [&]{
            inst_.reset(new Config(path));
        });
        return *inst_;
    }

    // 测试或重置用：显式销毁
    static void destroy() {
        // 注意：如果有并发使用者，自行保证时序
        inst_.reset();
        init_flag_ = std::once_flag{}; // 允许下次重新初始化
    }

    // 亦可提供测试注入入口：
    static void set_for_test(std::unique_ptr<Config> mock) {
        inst_ = std::move(mock);
        init_flag_ = std::once_flag{};
    }

    // 业务函数……
    std::string get(const std::string& key) const { /* ... */ return {}; }

    Config(const Config&) = delete;
    Config& operator=(const Config&) = delete;
    Config(Config&&) = delete;
    Config& operator=(Config&&) = delete;

private:
    explicit Config(const std::string& path) { /* 从 path 加载 */ }
    ~Config() = default;

    // C++17 起可用 inline 静态数据成员，避免在 .cpp 里再定义
    inline static std::unique_ptr<Config> inst_;
    inline static std::once_flag init_flag_;
};
```

**为什么这样写？**

- `std::call_once` 保证**只构造一次**，而且你能在**第一次**传入构造参数。
    
- 用 `std::unique_ptr` 管理生命周期，可选择 `destroy()` 主动销毁（如需要重载配置、或在单元测试中清理）。
    
- C++17 的 `inline static` 避免 ODR 额外定义，简化链接与跨 TU 使用。
# 常见坑与注意事项

- **静态初始化顺序灾难**：不同源文件里的全局/命名空间静态对象初始化顺序不确定，彼此互相依赖容易崩。用**“按需构造的局部静态”**可规避。
    
- **析构顺序**：程序退出时静态对象按**逆初始化顺序**析构；若退出阶段仍可能被其他静态对象调用，考虑“**不析构**”单例（泄露）或控制访问时序。
    
- **动态库/插件场景**：每个动态库可能各自有一份静态区，导致“多份单例”。若跨 DLL 共享，请把单例放到**唯一归属的模块**或经由**明确定义的接口**传递。
    
- **可测试性**：单例容易变“隐式全局依赖”，测试替换困难。若有测试需求，采用**②的可注入/可重置**版本，或干脆**避免单例，使用依赖注入（DI）**。
    
- **异常与重入**：若第一次构造抛异常，局部静态会在下一次再次尝试构造（标准保证）；自己写 `call_once` 也要注意异常会“没记住成功”，下一次仍会重试。
    
- **并发访问成本**：Meyers 单例的加锁只发生在**第一次初始化**；之后访问是普通引用返回，基本零开销。
    

# 何时不该用单例

- 需要在不同上下文创建多份隔离实例（如多数据库连接集）。
    
- 影响可测试性与可扩展性的大型系统。
    
- 只是为了“图方便”的全局变量替代品——这往往会让依赖关系变隐蔽。
    

# 小结（选型建议）

- **能用就用 ① Meyers**：最简单、线程安全、避免顺序灾难。
    
- **需要参数/重置/测试替换 → 用 ② `call_once` + `unique_ptr`**。

**为什么这样实现（要点）**

- Meyers：**按需构造**避开静态初始化顺序灾难；C++11 保证局部静态**一次性线程安全**；后续访问零锁开销。

- DCLP：用 `atomic` 的 acquire/release 防止“半初始化可见”；但相比前两种更易错，除非历史原因不建议首选。

```C++
    // ① 单例正常使用
    Logger::instance().log("Hello Singleton");

    std::cout << "cfg.path=" << cfg.get("path") << "\n";

    // ② 并发获取应返回同一地址
    std::vector<const void*> addrs;
    std::mutex m;
    std::vector<std::thread> ts;
    for (int i = 0; i < 16; ++i) {
        ts.emplace_back([&]{
            auto& l = Logger::instance();
            std::lock_guard<std::mutex> lk(m);
            addrs.push_back(&l);
        });
    }
    for (auto& t : ts) t.join();
    bool all_same = std::all_of(addrs.begin()+1, addrs.end(),
```

# 4) 速查：选型建议

- **能用就用**：Meyers（简单、快、稳）。
    
- **需要首次传参/可重置/测试替换**：`SingletonOnce<T>`。
    
- **退出阶段还有被动访问且怕析构顺序问题**：`SingletonLeaky<T>`。
    
- **历史包袱/ABI 限制才考虑**：DCLP。
    

# 5) 常见坑（一句话版）

- **静态初始化顺序灾难**：用“按需构造”的函数内静态或 `call_once`。
    
- **析构顺序**：必要时用 “Leaky” 变体避免退出阶段访问崩溃。
    
- **跨 DLL/SO**：每个模块可能各自一份单例；要么放到统一模块，要么以接口传递实例。
    
- **可测试性**：若为全局依赖，优先用 `SingletonOnce` 的 **set_for_test / destroy**。

## 整合单例
### 单头文件 ：`singleton_all.hpp`
```C++
// singleton_all.hpp
#pragma once
#include <mutex>
#include <memory>
#include <utility>
#include <atomic>
#include <new>     // placement new for once_flag reset

// =============================
// ① Meyers Singleton（最简，线程安全）
// 说明：T 需可默认构造（可在 T 内部将构造设为私有，并 friend 本模板）
// =============================
template<class T>
struct SingletonMeyers {
    static T& instance() {
        static T inst;   // C++11 起函数内静态初始化线程安全
        return inst;
    }
    SingletonMeyers() = delete;
};

// ==========================================
// ② 可传参/可重置：std::call_once + std::unique_ptr
//  - 第一次调用可传 T 的构造参数
//  - destroy() 可显式销毁实例并复位（需自行保证期间无并发使用者）
//  - set_for_test() 可注入替身实例（测试友好）
// ==========================================
template<class T>
struct SingletonOnce {
    template<class... Args>
    static T& instance(Args&&... args) {
        std::call_once(flag_(), [&]{
            inst_().reset(new T(std::forward<Args>(args)...));
        });
        return *inst_();
    }
    static void destroy() {
        // 调用者需保证此时没有并发使用者
        inst_().reset();
        reset_flag_();
    }
    static void set_for_test(std::unique_ptr<T> mock) {
        // 调用者需保证此时没有并发使用者
        inst_() = std::move(mock);
        reset_flag_();
    }
private:
    // C++11 兼容：用函数内静态，而不是 inline static 成员
    static std::unique_ptr<T>& inst_() {
        static std::unique_ptr<T> p;
        return p;
    }
    static std::once_flag& flag_() {
        static std::once_flag f;
        return f;
    }
    static void reset_flag_() {
        // 原地析构并重建 once_flag，实现“复位”
        auto& f = flag_();
        f.~once_flag();
        new (&f) std::once_flag();
    }
};

// =======================================
// ②' Leaky 变体：有意不析构，避免退出次序问题（服务进程常用）
// =======================================
template<class T>
struct SingletonLeaky {
    static T& instance() {
        static T* p = new T(); // 故意不 delete；进程退出由 OS 回收
        return *p;
    }
    SingletonLeaky() = delete;
};

// =======================================
// ③ DCLP（不推荐，留作兼容）：atomic + 双检锁
// =======================================
template<class T>
struct SingletonDCLP {
    static T& instance() {
        T* p = ptr_().load(std::memory_order_acquire);
        if (!p) {
            std::lock_guard<std::mutex> lk(m_());
            p = ptr_().load(std::memory_order_relaxed);
            if (!p) {
                p = new T();
                ptr_().store(p, std::memory_order_release);
            }
        }
        return *p;
    }
private:
    static std::atomic<T*>& ptr_() {
        static std::atomic<T*> p{nullptr};
        return p;
    }
    static std::mutex& m_() {
        static std::mutex m;
        return m;
    }
};
```

```C++
    Logger& operator=(const Logger&) = delete;
    Logger(Logger&&) = delete;
    Logger& operator=(Logger&&) = delete;

private:
    friend struct SingletonMeyers<Logger>; // 允许模板访问私有构造/析构
    Logger()  { file_ = std::fopen("app.log", "a"); }
    ~Logger() { if (file_) std::fclose(file_); }

    std::FILE* file_{nullptr};
};
```

### 示例 2：`config.hpp`（`call_once` 可传参/可重置）
```C++
// config.hpp
#pragma once
#include "singleton_all.hpp"
#include <string>
#include <unordered_map>
#include <memory>

class Config {
public:
    // 第一次可传 path；之后忽略传参（如需重置请先 destroy）
    static Config& instance(const std::string& path = "config.json") {
        return SingletonOnce<Config>::instance(path);
    }
    static void destroy() { SingletonOnce<Config>::destroy(); }
    static void set_for_test(std::unique_ptr<Config> mock) {
        SingletonOnce<Config>::set_for_test(std::move(mock));
    }

    std::string get(const std::string& key) const {
        auto it = kv_.find(key);
        return it == kv_.end() ? std::string{} : it->second;
    }

    Config(const Config&) = delete;
    Config& operator=(const Config&) = delete;
    Config(Config&&) = delete;
    Config& operator=(const Config&&) = delete;

private:
    friend struct SingletonOnce<Config>;
    friend struct std::default_delete<Config>; // 允许 unique_ptr 默认删除器调用私有析构

    explicit Config(const std::string& path) {
        kv_["path"] = path;
        kv_["env"]  = "prod";
    }
    ~Config() = default;

    std::unordered_map<std::string, std::string> kv_;
};
```

```C++
    std::vector<const void*> addrs;

    std::vector<std::thread> ts;
    for (int i = 0; i < 16; ++i) {

            addrs.push_back(&l);
        });
    }
    for (auto& t : ts) t.join();
    bool all_same = std::all_of(addrs.begin()+1, addrs.end(),
        [&](auto p){ return p == addrs[0]; });

}
```
