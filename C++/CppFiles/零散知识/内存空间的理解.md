# 🧠【一张图：典型 C++ 程序的虚拟内存空间全貌】

下面是一个典型 64-bit 进程的虚拟地址空间，从**最高地址**到**最低地址**：
```C++
┌─────────────────────────────────────────────┐ 0xFFFFFFFFFFFFFFFF
│                                             │
│   内核空间（Kernel Space）                  │   仅供内核访问（用户态不能碰）
│   （在 Linux 一般是高地址 ~128TB 范围）     │
│                                             │
├─────────────────────────────────────────────┤
│                                             │
│   用户态保留区（Guard / 禁止访问段）         │
│                                             │
├─────────────────────────────────────────────┤ 高地址
│                栈（Stack）                  │
│   ↓ 向下生长（高地址 → 低地址）             │
│   函数调用帧、局部变量、返回地址            │
├─────────────────────────────────────────────┤
│                                             │
│   内存映射区（Memory-mapped region, mmap） │
│   动态库（Libc、libstdc++）、文件映射       │
│   匿名映射（malloc 大块内存直接 mmap）      │
│                                             │
├─────────────────────────────────────────────┤
│                堆（Heap）                   │
│   ↑ 向上生长（低地址 → 高地址）             │
│   new/malloc 由堆分配（brk/sbrk）           │
├─────────────────────────────────────────────┤
│   BSS 段（未初始化的全局/静态变量）          │
│       如： int g; static int s;             │
├─────────────────────────────────────────────┤
│   DATA 段（初始化的全局/静态变量）          │
│       如： int g=5; static int s=3;         │
├─────────────────────────────────────────────┤
│   RO DATA（只读常量区）                     │
│       字符串字面量、某些 const、constexpr  │
│       如： "hello", const int x=10;        │
├─────────────────────────────────────────────┤
│   TEXT 段（代码段，可执行指令）             │
│       二进制指令（函数代码）                │
└─────────────────────────────────────────────┘ 0x0000000000400000


```
这是一个非常典型的内存分区图，你在网上看到的各种图都是它的简化版。

# ⭐【对每个区域进行深入解释（从低地址到高地址）】

---

## 1️⃣ **TEXT 段（代码段）**

包含：

- 可执行机器指令（程序代码）
    
- 类成员函数、普通函数的最终机器码
    
- 一般是只读，有“可执行权限”
    

示例：
```C++
int add(int a, int b) { return a + b; }  // 这一段代码就在 TEXT
```
## 2️⃣ **RODATA 段（只读数据）**

包含：

- 字符串字面量
    
- const/constexpr 全局变量（常见编译器放这）
    
- vtables 常常也放在这里
    

示例：
```C++
const int x = 42;      // 通常放 rodata
const char* p = "Hi";  // "Hi" 在 rodata
```

## 3️⃣ **DATA 段（已初始化的全局/静态变量）**

包含：

- 已初始化的全局变量
    
- 已初始化的 static 变量
    

示例：
```C++
int g = 10;           // data
static int s = 20;    // data
```
## 4️⃣ **BSS 段（未初始化的全局/静态变量）**

包含：

- 未初始化的全局变量
    
- 初始化为 0 的静态变量
    

示例：
```C++
int g2;               // bss
static int s2;        // bss
```
## 5️⃣ **堆（Heap）**

动态内存区，特点：

- 向 **上** 生长
    
- 由 `malloc/new` 分配
    
- 大块分配可能直接使用 `mmap`
    
- 内存由程序员释放，出错容易（内存泄漏、悬挂指针）
    

示例：
```C++
int* p = new int(5);  // p 指向的 int 在堆上
```

## 6️⃣ **内存映射区（Memory-mapped region, mmap 区域）**

包含：

- 动态链接库（libc, libstdc++, .so/.dll）
    
- 文件映射（如 mmap 打开的文件）
    
- 共享内存（shared memory）
    
- malloc 使用的大块 mmap 区域
    

它位于堆和栈之间。

例子：
```C++
mmap(...)     // 文件映射区
dlopen(...)   // 动态库加载
```

## 7️⃣ **栈（Stack）**

特点：

- 向 **下** 生长（从高地址往低地址）
    
- 每次函数调用都会创建一个“栈帧”
    
- 局部变量、参数、返回地址都放栈里
    

示例：
```C++
void f(int x) {
    int y = 3;   // y 在栈中
}
```
## 8️⃣ **保留区 / 不可访问区**

系统用来保护栈和 mmap 区域，使越界访问立刻崩溃（如栈保护页）。

---

## 9️⃣ **内核空间（Kernel Space）**

用户程序不能访问（否则段错误），包括：

- 内核代码
    
- 设备驱动
    
- 内核数据结构
    

现代系统将虚拟空间顶部 128TB 或更大留作内核使用。
