# 核心概念解析：Agent、MCP、Skills、API、Plugin

## 一、概念总览与关系

```
┌─────────────────────────────────────────────────────────────┐
│                     Claude Code CLI                         │
│                                                             │
│  用户 --> /slash-command ──> [Skills/Commands]               │
│            |                      |                         │
│            v                      v                         │
│       Claude API  <── system prompt 注入                    │
│            |                                                │
│            v                                                │
│       tool_use 响应 ──> 工具调度器                           │
│            |              |          |          |           │
│            v              v          v          v           │
│       内置工具        MCP 工具    SubAgent    Hooks         │
│    (Read/Write/       (外部服务   (子代理)   (生命周期      │
│     Bash/Grep)        器提供)               回调)          │
│                           |                                 │
│                           v                                 │
│                      MCP Server                             │
│                   (独立进程/远程服务)                         │
│                                                             │
│  ┌──────────────────────────────────────────────────┐      │
│  │                  Plugin 系统                      │      │
│  │  打包 = Commands + Agents + Skills + Hooks + MCP  │      │
│  └──────────────────────────────────────────────────┘      │
└─────────────────────────────────────────────────────────────┘
```

**一句话总结各概念关系：**

> **API** 是底层通信协议；**Agent** 是 Claude Code 的核心执行模式；**MCP** 是连接外部工具的标准协议；**Skills/Commands** 是用户扩展 Claude 行为的指令模板；**Plugin** 是将以上所有扩展打包分发的容器。

## 二、Agent（代理）

### 2.1 什么是 Agent

Agent 是一种**自主循环执行**的 AI 程序模式。Claude Code 本身就是一个 Agent -- 它接收目标，自主规划步骤，调用工具执行，根据结果调整策略，直到任务完成。

### 2.2 Claude Code 中的 Agent 层级

```
主 Agent（Claude Code 主会话）
    |
    +-- SubAgent（子代理，通过 Task 工具创建）
    |       |
    |       +-- 独立上下文窗口
    |       +-- 可选择的工具子集
    |       +-- 完成后返回结果给父级
    |
    +-- Agent Teams（实验性，多个 Builder 并行工作）
            |
            +-- Lead Agent（协调者）
            +-- Builder 1（写文件 A, B）
            +-- Builder 2（写文件 C, D）
```

### 2.3 子代理类型

通过 `Task` 工具的 `subagent_type` 参数指定：

| 类型 | 用途 | 可用工具 |
|------|------|----------|
| **Explore** | 代码库探索、文件查找 | Read, Glob, Grep, WebFetch, WebSearch |
| **Plan** | 方案规划、架构设计 | Read, Glob, Grep, WebFetch, WebSearch（不含写工具） |
| **Bash** | 命令执行 | Bash |
| **general-purpose** | 通用任务 | 所有工具 |
| **自定义 Agent** | 用户定义的专用代理 | frontmatter 中指定 |

### 2.4 自定义 Agent

定义在 `~/.claude/agents/` 或 `.claude/agents/` 目录下：

```markdown
---
name: planner
description: 任务规划师 - 使用 WBS 方法论分解功能需求
tools: Read, Write, mcp__contextweaver__codebase-retrieval
color: blue
---

# Agent 系统提示词

你是一个任务规划专家...
```

**frontmatter 字段：**

| 字段 | 说明 | 示例 |
|------|------|------|
| `name` | Agent 标识名 | `planner` |
| `description` | 描述（供 Claude 判断何时使用） | `任务规划师` |
| `tools` | 可使用的工具白名单（逗号分隔） | `Read, Write, Grep` |
| `color` | 终端显示颜色 | `blue`, `cyan`, `magenta` |
| `model` | 使用的模型（可选） | `sonnet`, `opus`, `haiku` |

### 2.5 Agent 执行原理

```
Task(subagent_type="Explore", prompt="查找认证相关代码")
    |
    v
创建新的 Claude API 会话（独立上下文）
    |
    v
注入子代理的系统提示词 + 工具定义
    |
    v
子代理自主循环（读文件、搜索、分析...）
    |
    v
返回最终结果给主 Agent
    |
    v
主 Agent 继续后续工作
```

## 三、MCP（Model Context Protocol）

### 3.1 什么是 MCP

MCP 是 Anthropic 设计的**开放标准协议**，用于标准化 AI 应用程序与外部工具/数据源之间的连接。常被比喻为 **"AI 的 USB-C"** -- 一个统一接口连接一切。

**核心架构：**

```
┌─────────────┐     JSON-RPC 2.0     ┌─────────────┐
│  MCP Host    │ <==================> │  MCP Server  │
│ (Claude Code)│                      │ (外部工具)    │
│              │  stdio / SSE / HTTP  │              │
│  MCP Client  │ <--- Transport --->  │  工具提供者   │
└─────────────┘                       └─────────────┘
```

### 3.2 MCP Server 暴露的三种原语

| 原语 | 说明 | 示例 |
|------|------|------|
| **Tools（工具）** | 模型可调用的函数 | 搜索代码、查询数据库、浏览器操作 |
| **Resources（资源）** | 可读取的数据源 | 文件内容、数据库记录、API 数据 |
| **Prompts（提示）** | 预定义的提示模板 | 代码审查模板、分析模板 |

### 3.3 传输类型

| 传输方式 | 通信方式 | 适用场景 | 配置示例 |
|----------|----------|----------|----------|
| **stdio** | 子进程 stdin/stdout | 本地工具（最常用） | `{"command": "npx", "args": [...]}` |
| **SSE** | HTTP + Server-Sent Events | 云端服务 | `{"type": "sse", "url": "https://..."}` |
| **HTTP** | REST 请求/响应 | 无状态 API | `{"type": "http", "url": "https://..."}` |
| **WebSocket** | 双向实时通信 | 实时服务 | `{"type": "ws", "url": "wss://..."}` |

### 3.4 配置方式

**在 settings.json 中直接配置：**

```json
{
  "mcpServers": {
    "context7": {
      "command": "npx",
      "args": ["-y", "@upstash/context7-mcp"]
    },
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    }
  }
}
```

**通过 CLI 管理：**

```bash
claude mcp add context7 npx -y @upstash/context7-mcp        # 添加
claude mcp add --global deepwiki npx -y mcp-deepwiki         # 添加全局
claude mcp remove context7                                    # 移除
claude mcp list                                               # 列出所有
```

**通过插件的 .mcp.json 文件：**

```json
{
  "mcp-deepwiki": {
    "command": "npx",
    "args": ["-y", "mcp-deepwiki"]
  }
}
```

### 3.5 工具注册流程

```
Claude Code 启动
     |
     v
加载 settings.json + 插件的 .mcp.json
     |
     v
启动/连接各 MCP Server
     |
     v
向每个 Server 发送 tools/list 请求
     |
     v
Server 返回工具定义列表
     |
     v
工具以 mcp__<服务器名>__<工具名> 格式注册
     |
     v
合并到 Claude 可用工具列表中
```

### 3.6 创建自定义 MCP Server

**TypeScript 示例（使用 @modelcontextprotocol/sdk）：**

```typescript
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { z } from "zod";

const server = new McpServer({
  name: "my-server",
  version: "1.0.0",
});

// 注册工具
server.tool(
  "search_files",
  "Search files in a directory",
  {
    query: z.string().describe("Search query"),
    path: z.string().optional().describe("Directory path"),
  },
  async ({ query, path }) => {
    const results = await doSearch(query, path);
    return {
      content: [{ type: "text", text: JSON.stringify(results) }],
    };
  }
);

// 启动 stdio 传输
const transport = new StdioServerTransport();
await server.connect(transport);
```

**Python 示例（使用 mcp 包）：**

```python
from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp.types import Tool, TextContent

app = Server("my-server")

@app.list_tools()
async def list_tools():
    return [
        Tool(
            name="search_files",
            description="Search files in a directory",
            inputSchema={
                "type": "object",
                "properties": {
                    "query": {"type": "string"},
                },
                "required": ["query"],
            },
        )
    ]

@app.call_tool()
async def call_tool(name: str, arguments: dict):
    if name == "search_files":
        results = search_files(arguments["query"])
        return [TextContent(type="text", text=str(results))]

async def main():
    async with stdio_server() as (read_stream, write_stream):
        await app.run(read_stream, write_stream)
```

### 3.7 常见 MCP Server

| 服务器 | 用途 | 安装命令 |
|--------|------|----------|
| `@upstash/context7-mcp` | 库文档查询 | `npx -y @upstash/context7-mcp` |
| `@playwright/mcp` | 浏览器自动化 | `npx @playwright/mcp@latest` |
| `mcp-deepwiki` | 开源项目文档 | `npx -y mcp-deepwiki` |
| `@anthropic-ai/mcp-server-sequential-thinking` | 深度推理 | `npx -y @anthropic-ai/...` |
| `@modelcontextprotocol/server-filesystem` | 文件系统 | `npx -y @modelcontextprotocol/server-filesystem` |

## 四、Skills 与 Commands（技能与命令）

### 4.1 概念区分

Claude Code 有三种扩展机制，触发方式不同：

| 类型 | 定义位置 | 触发方式 | 触发者 |
|------|----------|----------|--------|
| **Commands（斜杠命令）** | `commands/` 目录 | 用户手动输入 `/command` | 用户 |
| **Skills（技能）** | `skills/` 目录 | Claude 根据上下文自动判断 | 模型 |
| **Agents（代理）** | `agents/` 目录 | 在 Skill/Command 中通过 Task 调用 | 模型 |

### 4.2 Commands（斜杠命令）

**文件位置与命名规则：**

```
# 用户全局命令
~/.claude/commands/commit.md           --> /commit
~/.claude/commands/ccg/workflow.md     --> /ccg:workflow
~/.claude/commands/zcf/feat.md         --> /zcf:feat

# 项目级命令
.claude/commands/deploy.md             --> /project:deploy
.claude/commands/ops/migrate.md        --> /project:ops:migrate
```

**文件格式（Markdown + YAML frontmatter）：**

```markdown
---
description: '智能 Git 提交：分析改动生成 Conventional Commit 信息'
allowed-tools: [Bash, Read, Grep]
---

# Commit - 智能 Git 提交

用户参数：$ARGUMENTS

## 执行步骤

1. 运行 `git status` 检查仓库状态
2. 运行 `git diff --staged` 分析暂存改动
3. 生成 Conventional Commits 格式的提交信息
4. 执行 `git commit`
```

**frontmatter 字段：**

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| `description` | 命令描述，显示在 `/help` 中 | 是 |
| `allowed-tools` | 预授权工具列表（减少确认弹窗） | 否 |
| `argument-hint` | 参数提示 | 否 |
| `model` | 覆盖模型选择 | 否 |

**内置变量：**
- `$ARGUMENTS` -- 用户在命令后输入的参数文本

**内置命令：**

| 命令 | 功能 |
|------|------|
| `/help` | 显示帮助信息 |
| `/clear` | 清除对话历史 |
| `/compact` | 压缩/总结对话 |
| `/config` | 管理配置 |
| `/cost` | 显示 token 使用量 |
| `/init` | 初始化 CLAUDE.md |
| `/model` | 切换模型 |
| `/permissions` | 管理工具权限 |
| `/memory` | 管理 CLAUDE.md |
| `/vim` | 切换 vim 输入模式 |
| `/fast` | 切换快速模式 |

### 4.3 Skills（技能）

Skills 与 Commands 的关键区别是**触发方式** -- Skills 由 Claude 模型自动判断激活。

**定义格式（`skills/<skill-name>/SKILL.md`）：**

```markdown
---
name: code-review
description: 当用户请求代码审查或提到代码质量时自动激活
version: 1.0.0
---

# 代码审查技能

当用户的任务涉及代码审查时，按照以下步骤执行...
```

**目录结构：**

```
skills/
└── code-review/
    ├── SKILL.md          # 主定义文件（必需）
    ├── references/       # 参考资料（可选）
    ├── examples/         # 示例（可选）
    └── scripts/          # 辅助脚本（可选）
```

### 4.4 Commands vs Skills 对比

| 维度 | Commands | Skills |
|------|----------|--------|
| 触发 | 用户手动 `/command` | Claude 自动判断 |
| 优势 | 精确控制、参数传递 | 无缝集成、自动化 |
| 场景 | 明确的操作流程（git commit） | 上下文感知的能力增强 |
| description 作用 | 帮助信息展示 | **决定是否激活的关键依据** |

## 五、API（Claude API）

### 5.1 Claude Code 与 API 的关系

Claude Code 是 Claude API 的**高级封装客户端**。它将终端交互、文件系统操作、工具调度等能力封装在 API 调用之上。

```
用户操作 --> Claude Code CLI --> Messages API --> Claude 模型
                                     |
                              Tool Use（函数调用）
                                     |
                              工具执行 + 结果回传
```

### 5.2 Messages API 核心要素

| 要素 | 说明 |
|------|------|
| `model` | 使用的模型（claude-opus-4-20250514 等） |
| `system` | 系统提示词 |
| `messages` | 对话消息数组（user/assistant 交替） |
| `tools` | 可用工具定义数组 |
| `tool_choice` | 工具选择策略（auto/any/specific） |
| `stream` | 是否流式输出 |
| `max_tokens` | 最大输出 token 数 |

### 5.3 Tool Use 机制

这是 Claude Code 的核心支柱 -- Claude 模型原生支持 **Function Calling**：

1. 请求中定义 `tools`（工具名 + 描述 + JSON Schema 参数）
2. Claude 返回 `tool_use` 内容块（工具名 + 参数）
3. 客户端执行工具，返回 `tool_result`
4. Claude 基于结果继续推理
5. 循环直到 Claude 不再调用工具

### 5.4 流式响应

Claude Code 使用流式 API（SSE 格式）实时输出：

```
event: message_start      --> 消息开始
event: content_block_start --> 内容块开始（text 或 tool_use）
event: content_block_delta --> 增量内容
event: content_block_stop  --> 内容块结束
event: message_stop        --> 消息结束
```

## 六、Plugin（插件）

### 6.1 什么是 Plugin

Plugin 是 Claude Code 的**扩展打包与分发机制**。一个插件可以包含 Commands、Agents、Skills、Hooks、MCP Server 的任意组合。

### 6.2 插件目录结构

```
plugin-name/
├── .claude-plugin/
│   └── plugin.json          # 插件元数据（必需）
├── commands/                # 斜杠命令（可选）
│   └── my-command.md
├── agents/                  # Agent 定义（可选）
│   └── my-agent.md
├── skills/                  # 技能定义（可选）
│   └── my-skill/
│       └── SKILL.md
├── hooks/                   # Hook 配置（可选）
│   └── hooks.json
├── .mcp.json                # MCP 服务器配置（可选）
├── scripts/                 # 辅助脚本（可选）
└── README.md                # 文档
```

### 6.3 plugin.json 格式

```json
{
  "name": "my-plugin",
  "version": "1.0.0",
  "description": "插件描述",
  "author": "作者名"
}
```

### 6.4 插件安装位置

```
~/.claude/plugins/
├── marketplaces/
│   └── claude-plugins-official/     # 官方插件市场
│       ├── plugins/                 # 内置插件
│       │   ├── security-guidance/
│       │   ├── hookify/
│       │   └── ...
│       └── external_plugins/        # 外部集成
│           ├── playwright/
│           ├── context7/
│           └── deepwiki/
└── local/                           # 本地插件
```

### 6.5 插件管理

```bash
/plugin install <name>     # 安装插件
/plugin remove <name>      # 移除插件
/plugin list               # 列出已安装插件
/plugin discover           # 浏览可用插件
```

## 七、Hooks（钩子）

### 7.1 什么是 Hooks

Hooks 是 Claude Code 生命周期中的**确定性回调点**。在特定事件发生时执行自定义 Shell 命令，实现对 Claude 行为的细粒度控制。

### 7.2 事件类型

| 事件 | 触发时机 | 典型用途 |
|------|----------|----------|
| `PreToolUse` | 工具执行之前 | 安全检查、参数验证、拦截 |
| `PostToolUse` | 工具执行之后 | 日志记录、后处理、通知 |
| `Stop` | Claude 完成回复时 | 自动继续循环、清理 |
| `UserPromptSubmit` | 用户提交消息时 | 输入预处理、上下文注入 |
| `SessionStart` | 会话启动时 | 环境初始化 |
| `Notification` | 发送通知时 | 自定义通知方式 |

### 7.3 配置格式

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Edit|Write|MultiEdit",
        "hooks": [
          {
            "type": "command",
            "command": "python3 /path/to/security_check.py",
            "timeout": 10
          }
        ]
      }
    ],
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "/path/to/on_stop.sh"
          }
        ]
      }
    ]
  }
}
```

### 7.4 Hook 脚本交互

- **输入**：通过 stdin 接收 JSON（包含工具名、参数等上下文）
- **输出**：通过 stdout 返回 JSON 响应
  - `{"decision": "allow"}` --> 继续执行
  - `{"decision": "block", "reason": "..."}` --> 阻止执行
  - `{"decision": "modify", ...}` --> 修改参数后执行

### 7.5 可用环境变量

| 变量 | 说明 |
|------|------|
| `${CLAUDE_PLUGIN_ROOT}` | 插件根目录（仅插件 hooks） |
| `${CLAUDE_PROJECT_DIR}` | 当前项目目录 |

## 八、概念关系总结

```
                    ┌─────────────┐
                    │  Claude API  │  <-- 底层通信协议
                    │ (Messages   │
                    │  API + Tool │
                    │  Use)       │
                    └──────┬──────┘
                           |
                    ┌──────v──────┐
                    │ Claude Code  │  <-- Agent 式 CLI 客户端
                    │ (主 Agent)   │
                    └──────┬──────┘
                           |
         ┌─────────┬───────┼────────┬──────────┐
         |         |       |        |          |
    ┌────v───┐ ┌───v──┐ ┌──v──┐ ┌──v───┐ ┌───v───┐
    │SubAgent│ │ MCP  │ │Skill│ │ Hook │ │Plugin │
    │(Task)  │ │Server│ │/Cmd │ │      │ │       │
    └────────┘ └──────┘ └─────┘ └──────┘ └───────┘
         |         |       |        |          |
         |    外部工具  提示词注入  生命周期   打包容器
         |    扩展      行为定制   拦截控制   (含以上所有)
    独立上下文
    并行执行
```

| 概念 | 本质 | 核心作用 |
|------|------|----------|
| **API** | HTTP 通信协议 | Claude Code 与 Claude 模型之间的桥梁 |
| **Agent** | 自主循环执行模式 | Claude Code 的核心执行架构 |
| **MCP** | 工具连接标准协议 | 让 Claude 使用任意外部工具 |
| **Skills/Commands** | Markdown 提示词模板 | 定制 Claude 的行为和工作流 |
| **Hooks** | 生命周期回调 | 确定性地拦截/监控/修改 Claude 行为 |
| **Plugin** | 扩展打包格式 | 将以上所有扩展组合分发 |
