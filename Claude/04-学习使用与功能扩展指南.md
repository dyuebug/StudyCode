# 学习使用与功能扩展指南

## 一、快速上手

### 1.1 安装

```bash
npm install -g @anthropic-ai/claude-code
```

### 1.2 认证

```bash
claude                 # 首次运行会引导认证
# 或设置 API Key
export ANTHROPIC_API_KEY=sk-ant-...
```

### 1.3 基本使用

```bash
# 交互模式
claude

# 单次模式
claude -m "帮我分析 src/index.ts 的代码结构"

# 管道模式
cat error.log | claude -m "分析这些错误日志"

# 恢复会话
claude --continue
claude --resume <session_id>
```

### 1.4 常用内置命令

| 命令 | 功能 |
|------|------|
| `/help` | 查看帮助 |
| `/clear` | 清空对话 |
| `/compact` | 压缩上下文 |
| `/model` | 切换模型 |
| `/cost` | 查看 token 消耗 |
| `/config` | 管理配置 |
| `/permissions` | 管理权限 |
| `/memory` | 管理 CLAUDE.md |
| `/init` | 初始化项目 CLAUDE.md |

## 二、配置体系

### 2.1 CLAUDE.md -- 自定义 Claude 行为

**全局配置（适用于所有项目）：**

创建 `~/.claude/CLAUDE.md`：

```markdown
Always respond in Chinese-simplified

# 我的全局规则

- 使用 TypeScript 编写代码
- 遵循 SOLID 原则
- 所有函数必须有 JSDoc 注释
```

**项目配置：**

在项目根目录创建 `CLAUDE.md` 或 `.claude/CLAUDE.md`：

```markdown
# 项目规则

## 技术栈
- Next.js 14 + TypeScript
- Tailwind CSS
- Prisma ORM

## 代码规范
- 组件使用 PascalCase 命名
- API 路由放在 src/app/api/ 下
- 所有数据库操作通过 Repository 层
```

### 2.2 settings.json -- 系统配置

**文件位置：** `~/.claude/settings.json`（全局）或 `.claude/settings.json`（项目）

```json
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "env": {
    "ANTHROPIC_BASE_URL": "https://api.anthropic.com",
    "MCP_TIMEOUT": "60000"
  },
  "permissions": {
    "allow": ["Bash", "Read", "Write", "Edit", "Glob", "Grep"],
    "deny": []
  },
  "model": "opus",
  "outputStyle": "engineer-professional",
  "includeCoAuthoredBy": false,
  "hooks": {},
  "mcpServers": {}
}
```

### 2.3 Output Style -- 自定义输出风格

**创建自定义输出风格：**

创建 `~/.claude/output-styles/my-style.md`：

```markdown
# 我的输出风格

## 语言
- 使用中文回复
- 技术术语保留英文

## 格式
- 代码变更前先说明原因
- 复杂修改使用表格展示变更点
- 始终包含代码引用（file:line）

## 行为
- 不主动提交 git
- 危险操作前确认
- 修改前先阅读现有代码
```

**启用：** 在 settings.json 中设置 `"outputStyle": "my-style"`

## 三、扩展功能

### 3.1 创建自定义 Command（斜杠命令）

**步骤 1：创建文件**

```
~/.claude/commands/my-review.md        --> /my-review
~/.claude/commands/team/standup.md     --> /team:standup
```

**步骤 2：编写内容**

```markdown
---
description: '智能代码审查 - 检查安全、性能和代码风格'
allowed-tools: [Bash, Read, Grep, Glob]
---

# 代码审查

## 审查目标

$ARGUMENTS

## 审查步骤

1. 运行 `git diff` 获取当前改动
2. 逐文件检查：
   - 安全性：SQL 注入、XSS、硬编码密钥
   - 性能：N+1 查询、不必要的重渲染
   - 代码风格：命名规范、DRY 原则
3. 输出结构化审查报告

## 输出格式

### Critical（必须修复）
...

### Warning（建议修复）
...

### Pass（通过）
...
```

**步骤 3：使用**

```
/my-review                          # 审查所有改动
/my-review feature/login-page       # 审查指定分支
```

### 3.2 创建自定义 Agent

**步骤 1：创建文件**

```
~/.claude/agents/my-architect.md
```

**步骤 2：编写内容**

```markdown
---
name: my-architect
description: 架构师 - 分析项目架构并提供改进建议
tools: Read, Glob, Grep
color: cyan
---

# 架构师 Agent

你是一个资深软件架构师。

## 职责

1. 分析项目结构和依赖关系
2. 识别架构问题和技术债
3. 提供可落地的改进方案

## 分析维度

- 模块划分是否合理
- 依赖方向是否正确
- 是否存在循环依赖
- 关注点是否分离
- 抽象层级是否恰当

## 输出格式

1. 架构现状总结
2. 问题清单（按严重程度排序）
3. 改进建议（含具体代码示例）
```

**注意：** Agent 由 Claude 在需要时自动通过 Task 工具调用，无法直接用斜杠命令触发。

```json
    }
  }
}
```

#### 方式 3：通过插件

安装包含 MCP Server 的插件，其 `.mcp.json` 会自动加载。

### 3.4 创建自定义 MCP Server

当内置工具和现有 MCP Server 无法满足需求时，可以自己编写。

**TypeScript 实现：**

```typescript
// my-server.ts
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { z } from "zod";

const server = new McpServer({
  name: "my-custom-tools",
  version: "1.0.0",
});

// 定义一个查询数据库的工具
server.tool(
  "query_database",
  "Execute a read-only SQL query against the project database",
  {
    sql: z.string().describe("The SQL query to execute (SELECT only)"),
    database: z.string().optional().describe("Database name"),
  },
  async ({ sql, database }) => {
    // 安全检查
    if (!sql.trim().toUpperCase().startsWith("SELECT")) {
      return {
        content: [{ type: "text", text: "Error: Only SELECT queries are allowed" }],
        isError: true,
      };
    }
    // 执行查询
    const results = await executeQuery(sql, database);
    return {
      content: [{ type: "text", text: JSON.stringify(results, null, 2) }],
    };
  }
);

// 启动
const transport = new StdioServerTransport();
await server.connect(transport);
```

**Python 实现：**

```python
# my_server.py
from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp.types import Tool, TextContent
import json

app = Server("my-custom-tools")

@app.list_tools()
async def list_tools():
    return [
        Tool(
            name="query_database",
            description="Execute a read-only SQL query",
            inputSchema={
                "type": "object",
                "properties": {
                    "sql": {"type": "string", "description": "SELECT query"},
                },
                "required": ["sql"],
            },
        )
    ]

@app.call_tool()
async def call_tool(name: str, arguments: dict):
    if name == "query_database":
        results = execute_query(arguments["sql"])
        return [TextContent(type="text", text=json.dumps(results))]

async def main():
    async with stdio_server() as (read_stream, write_stream):
        await app.run(read_stream, write_stream)

if __name__ == "__main__":
    import asyncio
    asyncio.run(main())
```

**注册使用：**

```json
{
  "mcpServers": {
    "my-tools": {
      "command": "node",
      "args": ["C:/path/to/my-server.js"]
    }
  }
}
```

### 3.5 配置 Hooks

**示例 1：安全检查（阻止危险 Bash 命令）**

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "python3 C:/Users/me/.claude/hooks/bash-guard.py",
            "timeout": 5
          }
        ]
      }
    ]
  }
}
```

```python
# bash-guard.py
import json, sys

data = json.load(sys.stdin)
command = data.get("input", {}).get("command", "")

dangerous = ["rm -rf", "DROP TABLE", "format", "del /s"]
for d in dangerous:
    if d.lower() in command.lower():
        print(json.dumps({
            "decision": "block",
            "reason": f"Blocked dangerous command containing '{d}'"
        }))
        sys.exit(0)

print(json.dumps({"decision": "allow"}))
```

**示例 2：自动日志记录**

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "python3 C:/Users/me/.claude/hooks/log-tool-use.py",
            "timeout": 5
          }
        ]
      }
    ]
  }
}
```

## 四、进阶使用技巧

### 4.1 多模型策略

```bash
# 在 settings.json 中设置默认模型
"model": "opus"

# 运行时切换
/model sonnet     # 切换到 Sonnet（更快更便宜）
/model opus       # 切换到 Opus（更强）
/fast             # 切换快速模式（同模型更快输出）
```

**建议策略：**

| 任务类型 | 推荐模型 | 原因 |
|----------|----------|------|
| 架构设计、复杂推理 | Opus | 最强推理能力 |
| 日常编码、重构 | Sonnet | 性价比最佳 |
| 简单查询、格式化 | Haiku | 速度快成本低 |

### 4.2 子代理高效使用

```
# 在对话中，Claude 会自动使用子代理
# 你也可以显式指导：

"请使用 Explore agent 查找所有与认证相关的文件"
"启动一个后台任务来运行测试"
"并行执行 lint 检查和类型检查"
```

### 4.3 权限预配置

在 `settings.json` 中预授权常用工具，减少交互确认：

```json
{
  "permissions": {
    "allow": [
      "Bash",
      "Read",
      "Write",
      "Edit",
      "Glob",
      "Grep",
      "WebSearch",
      "WebFetch",
      "mcp__context7",
      "mcp__mcp-deepwiki",
      "mcp__Playwright",
      "mcp__contextweaver"
    ]
  }
}
```

### 4.4 CI/CD 集成

```bash
# 在 CI 中使用 Claude Code
claude -m "检查这个 PR 是否有安全问题" \
  --allowedTools "Read,Grep,Glob" \
  --json \
  --output review.json

# 自动化测试生成
claude -m "为 src/utils.ts 生成单元测试" \
  --allowedTools "Read,Write,Bash" \
  --json
```

## 六、常见问题

### Q1：Claude Code 与 Cursor 等 IDE 工具有什么区别？

Claude Code 是终端优先的 Agent 式工具，强调自主执行和命令行集成；Cursor 等是 IDE 内的补全/辅助工具。Claude Code 更适合复杂的多步骤任务和自动化流程。

### Q2：MCP Server 运行失败怎么排查？

```bash
claude mcp list                    # 检查配置是否正确
claude /doctor                     # 运行诊断
# 检查 MCP_TIMEOUT 是否足够
# 确认 npx 和相关包是否可用
```

### Q3：上下文窗口不够用怎么办？

1. 使用 `/compact` 压缩对话
2. 使用子代理（Task）隔离上下文
3. 精确指定文件路径，避免读取不必要的文件
4. 使用 Grep/Glob 缩小搜索范围

### Q4：如何在团队中共享配置？

1. 将 `.claude/` 目录纳入版本控制（排除敏感信息）
2. 项目级 `CLAUDE.md` 定义团队规范
3. 项目级 `.claude/settings.json` 定义权限和 MCP
4. 共享 Plugin 统一工具链

### Q5：如何减少 token 消耗？

1. 切换到 Sonnet/Haiku 处理简单任务
2. 善用 `/compact` 压缩上下文
3. 精确描述需求，减少来回沟通
4. 使用子代理隔离大任务的上下文
5. 在 CLAUDE.md 中预置常用上下文，减少重复说明
