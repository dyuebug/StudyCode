# Claude Code 发展历程

## 一、Claude 模型发展线

Claude Code 建立在 Claude 模型家族之上，模型的演进直接影响 Claude Code 的能力边界。

### 1.1 Claude 模型时间线

| 时间 | 事件 | 意义 |
|------|------|------|
| **2023.03** | Claude 1.0 发布 | Anthropic 首个商用大语言模型 |
| **2023.07** | Claude 2 发布 | 200K 上下文窗口，能力大幅提升 |
| **2024.03** | Claude 3 系列发布 | Opus/Sonnet/Haiku 三级模型体系 |
| **2024.06** | Claude 3.5 Sonnet 发布 | 性价比标杆，编程能力突出 |
| **2024.10** | Claude 3.5 Sonnet (new) + Computer Use | 首次支持计算机操作能力 |
| **2025.02** | Claude 3.7 Sonnet 发布 | 混合推理模型，支持 extended thinking |
| **2025.04** | Claude 4 Sonnet 发布 | 更强的 Agent 能力 |
| **2025.05** | Claude 4 Opus 发布 | 旗舰模型，编程和推理能力巅峰 |
| **2025.xx** | Claude Opus 4.6 | 当前最新模型（本文撰写时使用的模型） |

## 二、MCP 协议发展线

| 时间 | 事件 | 意义 |
|------|------|------|
| **2024.11** | MCP 公开发布 | Anthropic 开源 MCP 规范和 SDK |
| **2024.11** | 首批 MCP Server 发布 | filesystem、GitHub、Slack 等参考实现 |
| **2024.12** | 社区快速采纳 | Block、Replit、Sourcegraph 等公司接入 |
| **2025.Q1** | MCP 生态爆发 | 数百个第三方 MCP Server 涌现 |
| **2025.Q2** | 规范持续演进 | 新增 HTTP Streamable 传输、OAuth 认证等 |
| **2025-2026** | 行业标准化 | MCP 成为 AI 工具连接的事实标准之一 |

**MCP 的设计哲学：**
- 被称为 **"AI 的 USB-C"** -- 用一个统一协议取代碎片化的集成方式
- 基于 JSON-RPC 2.0，轻量且易于实现
- 开源规范，鼓励社区贡献

## 三、Claude Code 发展线

### 3.1 核心里程碑

| 时间 | 版本/事件 | 关键特性 |
|------|-----------|----------|
| **2025.02** | Claude Code 首次发布（Research Preview） | 基础 CLI 工具：文件读写、Bash 执行、对话交互 |
| **2025.03-04** | 快速迭代期 | MCP 集成、权限系统、/compact 压缩、多模型支持 |
| **2025.05** | 重要功能更新 | Hooks 系统、自定义 Commands、Agent 子系统 |
| **2025.06-07** | 生态建设期 | Plugin 系统、Skills 机制、Agent Teams（实验性） |
| **2025.08-09** | 稳定与优化 | 性能优化、IDE 集成（VS Code/JetBrains/Neovim）增强 |
| **2025.10-12** | 成熟期 | Output Styles、Status Line、插件市场 |
| **2026.01-02** | 当前版本 | 完整的扩展生态、多模型协作、Agent Teams |

### 3.2 功能演进详情

#### 第一阶段：基础能力（2025.02-03）

- **核心工具**：Read、Write、Edit、Bash、Glob、Grep
- **对话模式**：交互式 REPL、单次模式（-m 参数）
- **基本配置**：CLAUDE.md 项目指令文件
- **API 集成**：直接调用 Anthropic Messages API

#### 第二阶段：工具生态（2025.03-05）

- **MCP 集成**：支持 stdio 传输的 MCP Server
- **权限系统**：allow/deny 白名单黑名单
- **上下文管理**：/compact 命令、自动压缩
- **多模型支持**：Opus/Sonnet/Haiku 切换
- **WebSearch/WebFetch**：内置网络访问能力

#### 第三阶段：扩展机制（2025.05-07）

- **Hooks 系统**：PreToolUse、PostToolUse、Stop 等生命周期钩子
- **自定义 Commands**：用户通过 Markdown 定义斜杠命令
- **子代理系统**：Task 工具、subagent_type（Explore/Plan/Bash）
- **Skills 机制**：模型自动激活的技能系统
- **自定义 Agents**：用户定义的专用代理
- **Plugin 系统**：标准化的扩展打包与分发
- **Agent Teams**（实验性）：多 Agent 并行协作

#### 第四阶段：生态成熟（2025.08-2026）

- **IDE 集成增强**：VS Code、JetBrains、Neovim 原生集成
- **Output Styles**：自定义 Claude 的输出风格和行为规范
- **Status Line**：自定义状态栏显示
- **插件市场**：官方插件市场与社区插件
- **SSE/HTTP 传输**：MCP 支持远程服务器
- **会话管理增强**：--resume、--continue、会话持久化
- **管道/CI 模式**：无人值守执行支持
- **多层级配置**：全局/项目/企业三级配置体系

## 四、行业背景与竞品对比

### 4.1 AI 编程工具发展线

| 时间 | 事件 | 影响 |
|------|------|------|
| **2021** | GitHub Copilot 发布 | AI 辅助编程时代开启 |
| **2023** | ChatGPT + Code Interpreter | AI 代码执行能力 |
| **2024** | Cursor / Aider / Continue 等 IDE 工具涌现 | AI 编程工具多元化 |
| **2024.11** | MCP 协议发布 | 工具连接标准化 |
| **2025.02** | Claude Code 发布 | Agent 式终端编程工具 |
| **2025.Q1** | OpenAI Codex CLI、Google Gemini CLI | 终端 AI 编程工具赛道形成 |
| **2025-2026** | 行业成熟 | 从辅助补全到自主 Agent 的范式转换 |

### 4.2 Claude Code 的差异化定位

| 维度 | Claude Code | 传统 AI 编程工具 |
|------|-------------|-----------------|
| **执行模式** | Agent 自主循环 | 单次补全/建议 |
| **工作环境** | 终端 CLI | IDE 插件 |
| **工具调用** | 原生 Tool Use | 有限的外部交互 |
| **扩展性** | MCP + Plugin + Hooks | 封闭或有限扩展 |
| **上下文** | 整个项目 + 多层级配置 | 当前文件/选中代码 |
| **协作方式** | 多 Agent 并行 | 单线程交互 |
